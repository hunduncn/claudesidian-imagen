# 使用指南

从**选文章**到**图片落地到 vault**，完整流程大概 1-2 分钟。

---

## 界面三栏

```
┌──────────┬──────────────────────┬──────────────────┐
│  文件树   │   正文预览 + 控件     │   生成结果       │
│  vault   │   类型 / 标题 / 风格   │   候选图 + 操作  │
│          │   / 额外指令 / 数量    │                 │
└──────────┴──────────────────────┴──────────────────┘
```

- **左栏**：vault 文件树，点文件夹展开，点 .md 打开
- **中栏**：文章预览 + 所有生成参数
- **右栏**：生成出来的候选图网格（生成前隐藏）

---

## 完整工作流

### 1. 选文章

左侧点一篇 `.md`，中栏会显示：
- 文章标题（H3）
- 正文预览（灰底区域）
- 下方出现三个类型按钮

### 2. 选类型

| 类型 | 比例 | 尺寸 | 有无文字 |
|---|---|---|---|
| 小红书封面 | 3:4 | 1242×1660 | ✅ 带标题 |
| 公众号封面 | 2.35:1 | 900×383 | ✅ 带标题（宽 banner） |
| 公众号插图 | 16:9 | 1280×720 | ❌ 纯视觉无文字 |

选完之后出现控件区。

### 3. 改标题（可选，仅两种带文字类型）

"封面标题文字" 输入框默认填入文章 H1（没 H1 就用文件名）。**会渲染到图里**。

**什么时候要改：**
- 原标题有触发安全分类器的词（"崩了" / "暴跌" / "诈骗" 等）
- 原标题太长，封面放不下
- 想换一句更吸引眼球的

改过之后下面会显示"已改写标题。[恢复原标题]"，点那个链接能一键还原。

### 4. 选风格家族

下拉里有 27 个风格：

**品牌就绪（带 ⭐）**——跨三种类型都有专门设计，优先选这三个：
- `business-clean` · 商务干净
- `editorial-minimal` · 编辑极简
- `knowledge-blogger` · 知识博主

**跨类型家族**（6 个）——任一类型都能选：
- `morandi` · 莫兰迪
- `new-chinese` · 新中式
- `film-cinematic` · 胶片
- `risograph` · 丝网印刷
- `oil-painterly` · 油画
- `magazine-collage` · 杂志拼贴

**类型专属**（剩下的）——只在该类型下显示。

### 5. 额外指令（可选）

一两句话影响最终效果，比如：
- `主色调用绿色`
- `画面里有一只猫`
- `背景加一些几何线条`

留空也行。

### 6. 选生成数量

1 / 2 / 3 / 4 张同时生成。刚尝试一个风格选 1 探路，风格合适再批量出 4 张挑。

### 7. 点"生成 N 张"

右栏会按选定数量显示占位，依次填入结果。每张 20-60 秒。

### 8. 挑一张 → 采纳

- **点图片**：选中（边框变蓝）
- **🔍**：放大预览，细节看清
- **🔗 设为基准**：锁定这张的风格给当前家族（下一节详细讲）
- **重试**：单张失败可以只重新生成这一张，不用全部重来

选中一张之后点"采纳并保存"。图会落到：

```
<vault>/05_Attachments/Organized/<文章名>/<类型>_v1.png
```

重复生成同文章同类型会自动递增版本号 `_v2`, `_v3`...

wikilink 自动复制到剪贴板：

```
![[05_Attachments/Organized/文章名/xhs-cover_v1.png]]
```

到笔记里直接 ⌘+V 粘贴即可。

## 进阶：品牌锚定（保持视觉一致）

### 是什么

对一个风格家族**钉一张基准图**，之后用这个家族生成任何图都会以这张为视觉参考——**构图、配色、材质、光感**都会被带向基准图。

适合场景：你在做一个公众号/小红书账号，想让所有封面有统一的识别度。

### 怎么设

1. 正常生成几张，挑一张你觉得"定调"的
2. 鼠标悬停在那张图上，点 🔗 **设为基准**
3. 右栏和下拉里的家族名称前会出现 📌 标记

### 效果

之后同一家族生成，自动把基准图附在请求里（multimodal），模型会维持相近的视觉风格——不是像素级复刻，是氛围上的延续。

### 怎么取消

下拉框选到该家族 → 下方出现"该家族已设参考基准" → 点"取消基准"。

## 界面行为小贴士

- **切换文章** = 清空生成结果 + 重置标题为新文章的 H1
- **切换类型** = 只清结果，标题改写保留
- **重新点同一篇文章** = 标题会被重置（这是"重新开始"，如果你在意，改完标题就直接生成，别乱点）
- **端口**：默认 5173，改 `~/.claudesidian-imagen/config.json` 的 `preferredPort`

## 典型使用节奏

**情景 A：日更公众号**

1. 写完一篇文章 → 开 imagen
2. 选文章 → 选"公众号封面"
3. 用你品牌锚定的家族（比如 `business-clean`）
4. 生成 1 张 → 不满意就重试 / 生成 3 张挑
5. 采纳，wikilink 粘贴到文章顶部

**情景 B：风格摸索阶段**

1. 选一篇代表性的文章
2. 轮流试不同家族，每个生成 1 张
3. 选最打动你的那张
4. 点 🔗 设为基准
5. 从此用这个家族固定下来

**情景 C：标题触发安全策略**

1. 生成失败，提示"模型拒绝了这次请求"
2. 把"封面标题文字"里的敏感词改掉（"崩了" → "卡住了"）
3. 再生成

## 故障排查

| 现象 | 诊断 | 解决 |
|---|---|---|
| 右边空白不出图 | 终端看 log 有没有报错 | 检查 API Key / 网络 / 平台余额 |
| 反复"模型拒绝" | 标题 / 额外指令里有敏感词 | 改写标题、清空额外指令、换风格 |
| 字出错 / 乱码 | 模型对长中文标题不稳定 | 缩短标题，或再生成几张挑对的 |
| 图保存后 wikilink 贴到笔记不显示 | Obsidian 可能没识别到 | 关闭笔记重开，或检查附件文件夹确实有图 |
| 设置里改了配置没生效 | 要保存再刷新 | 点保存按钮 → ⌘+R |

更多见 [部署指南 · 常见问题](DEPLOY.md#常见问题)。
